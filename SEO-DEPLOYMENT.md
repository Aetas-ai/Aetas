# SEO Deployment and Maintenance Guide

Use this checklist when the company website is ready to move to the production domain. The production domain and canonical website address are:

```text
https://aetas.ai
```

Do not submit a temporary Hostinger URL or preview domain to search engines.

## What the Website Already Provides

The production build includes the following technical SEO features:

- Unique page titles and descriptions through `src/layouts/Layout.astro`
- Canonical URLs that point to `https://aetas.ai`
- Organization, WebSite, and WebPage structured data
- Open Graph and social sharing metadata
- Search-engine crawl instructions in `public/robots.txt`
- An XML sitemap generated by `@astrojs/sitemap`
- `noindex` handling for form confirmation pages
- Responsive layouts and optimized WebP assets

Every production build generates:

```text
dist/sitemap-index.xml
dist/sitemap-0.xml
dist/robots.txt
```

The sitemap updates automatically when public Astro pages are added or removed. Routes that should not appear in search results must also be reviewed in `astro.config.mjs` and marked with the `noIndex` layout property.

## Before Production Deployment

1. Confirm the final company name, service descriptions, email address, and legal content.
2. Replace remaining placeholder copy and images.
3. Run the production build:

   ```sh
   npm install
   npm run build
   ```

4. Confirm the build completes without errors.
5. Open `dist/sitemap-0.xml` and confirm that it contains only public pages.
6. Confirm private, duplicate, draft, and confirmation pages are not listed.
7. Test the website on mobile, tablet, and desktop.

## Production Domain Checklist

After deploying the new build:

1. Confirm `https://aetas.ai` displays the new Aetas Global Innovation website.
2. Confirm these URLs load successfully:

   ```text
   https://aetas.ai/robots.txt
   https://aetas.ai/sitemap-index.xml
   https://aetas.ai/aetasIcon.webp
   ```

3. Confirm the website uses HTTPS without certificate warnings.
4. Choose one canonical domain format. This project uses `https://aetas.ai` without `www`.
5. Configure `http://aetas.ai` and `https://www.aetas.ai` to redirect permanently to `https://aetas.ai` when Hostinger or Cloudflare access is available.
6. Do not keep another public copy of the website on a temporary domain unless that copy is blocked from indexing.

## Google Search Console

1. Open [Google Search Console](https://search.google.com/search-console/).
2. Add `aetas.ai` as a **Domain property**.
3. Add the DNS TXT verification record provided by Google to the active DNS provider.
4. Wait for verification to complete. A TXT verification record does not redirect website or email traffic.
5. Open **Sitemaps** and submit:

   ```text
   https://aetas.ai/sitemap-index.xml
   ```

6. Use **URL Inspection** for `https://aetas.ai/` and select **Request indexing**.
7. Repeat URL Inspection for the main service pages if needed:

   ```text
   https://aetas.ai/ai/
   https://aetas.ai/security/
   https://aetas.ai/global/
   https://aetas.ai/about/
   ```

8. Review the Pages, Sitemaps, Core Web Vitals, and Enhancements reports after Google begins crawling the site.

## After Every Website Update

For the current Hostinger file-upload workflow:

1. Merge approved changes into the repository's main branch.
2. Download or prepare the latest project files.
3. Deploy the updated source package through the Hostinger Web App deployment.
4. Allow Hostinger to install dependencies and run the Astro production build.
5. Confirm the deployment completed successfully.
6. Test the changed pages on the production domain.
7. Recheck the sitemap when pages were added, renamed, or removed.
8. Request indexing in Search Console only for important new or substantially updated pages. It is not necessary for every small UI change.

Environment variables should be configured in Hostinger's application settings rather than committed to Git or included in uploaded ZIP files. Review this guide when the deployment workflow changes to Git-based deployment or CI/CD.

## Ongoing SEO Work

Technical SEO makes the site crawlable, but it does not guarantee first-place rankings. Continue with:

- Accurate, useful service-page content
- Consistent use of the company name **Aetas Global Innovation**
- Official profiles that link to `https://aetas.ai`
- Useful articles about AI workflow automation, cybersecurity, managed IT, helpdesk, and BPO operations
- Legitimate company mentions and backlinks
- Regular checks for broken links, missing pages, slow assets, and mobile usability problems
- Search Console review after major content or routing changes

Do not add fabricated customers, results, certifications, addresses, testimonials, or partner levels for SEO purposes.
